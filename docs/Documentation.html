What is TTML?

TTML is a shortform for (Taxi To Markup Language)

TTML is a lightweight language that compiles to Markup it extends normal markup language and providing a quicker and cleaner way to generate markup on the fly

Why TTML?
  • Easy to learn
  • Allows you to define your own tags like XML
  • Library of shortened HTML tags making your work easier (Shortcuts)
  • Easier ways of declaring common attributes eg:class, id,....
  • Allows imports from other files
  • Not dependent on indentation to code
  • Can produce almost any markup eg: HTML,XML,XHTML..
  • Saves time
  • Has extension where you can write your normal markup in your file
  • Produces indented code (depending on how you type your code + an option to join lines of your code)
  • ERB templating support
  • Saves space, 70% smaller than XML and 86% smaller than JSON holding the same data

Basic Things in using TTML

You need to at least basic experience in Markup language to use TTML
If you have this it will take less than thirty minutes to get started
This documentation was created using TTML by the way then converted to HTML
To see code add /ttml to any page url in this website eg: /documentation/ttml

Below we are going to introduce you to the basic syntax that you need to know:

Symbols

# => used to indicate you want the interpreter to ignore this line
! => Used for comments(TTML) this do not appear in your final product
% => used to ignore multiple lines until the next (%) which indicates the end of that section
{} => used for tags
& => used for imports
Lastly, between the exclamation marks we declare some common attributes class or id or href or src or just all of them, your choice.

We want to make it clear now that each element should go on a separate line no matter what even if it is a closing brace for tags that exceed a single line
Do not put TTML code inside another TTML brace unless on different lines eg: sp{a !@http://google.com!{google} }, this link will be treated as plain text hence not interpreted
For the above example if it is a must do this: sp{<a href="http://google.com">google</a>}, that will do it
Do not close a multi line tags behind another one eg: d {.........several lines sp{hello}}, the last brace should go on a separate line for your code to be properly executed
The output will only appear indented if your code was indented
Look at the syntax expressed in the examples below to understand what we mean
Avoid leaving blank lines in the middle of your code especially within tags
Actually the product code is indented so there is no need for blank lines
if an error occurs something like ......undefined method '[]'..., look for blank lines in your code(especially within tags) and remove them

Imagine that is all plus your Knowledge in Markup and other
If you feel uncomfortable just go to the menu and click Try it here! type and compile the code to see the results yourself

Imports handling

To import a file begin a new line with &X where X is the name of the file you want to import
For TTML files you do not need to add the file extension do &header to import header.ttml
The rules apply the same about #, % and %
If you want a TTML file to be executed import it outside the % and %
To import any other type of file eg a stylesheet, script file import it between % and %
check the example below:

If you are importing a file and the file imports other files, all imports are solved hence the product will contain all imported files
If the file you are importing also has imports and continuing you will get all the data from the other files and all their imports
example: if you file imports &help.ttml then this file imports &about.ttml you will get both help.ttml and about.ttml in your file 
!DOCTYPE html
html{
     #<!--Executing imports -->
     &header
     #<!--header.ttml is imported outside % signs hence executed to markup code -->
  body{
     section class="blogs"{
        &blog
     }
     footer{
      %
      &footer.html
      <!-- This is imported inside the % signs hence not executed just inserted the way it is -->
      %
     }
    #<!--importing php files-->
     %
      <?php
       &login.php
     ?>
    %
  }
  script type="text/javascript"{
   %
     &path/to/script.js
   %
 }
}

Having your own tags

TTML allows you to write your own tags like how you do with XML
This increases the efficiency in maintenance of your project as the tags are descriptive
But to use this feature you need basic knowledge in Ruby to be able to edit the Shortcuts module
Check the simple example below: 
#<!-- This is not XML but HTML -->
single_blog{
  blog_name class="text-center"{Growing trees}
  blog_release_date{13-8-2018}
  blog_likes{345}
  blog_comments{45}
}

Here is a simple explanation of how this example works
You can instruct the interpreter the following:
single_blog => div
blog_name => h3
blog_release => p
blog_likes|blog_comments => span
Whenever the compiler sees the tag <single_blog> it knows you indicate <div>
Hence it produces <div> in your HTML product when interpreted and respectively for all other examples
When editing you should supply the your tag and then the HTML tag it should represent

Attributes

TTML provides you with a comfortable way of declaring attributes for tags
You can declare them as normal HTML attributes or TTML attributes(limited)
TTML attributes are limited to four most used tags that we observed: class,id,href and src
This attributes are declared inside ! and ! signs and attributes separated by a comma
Check the example below: 
 #<!-- This is a mashup of everything -->
 d !top_sect,wide-sect,#real,@http://www.youtube.com,>one.mp3!{}

<!--This is a mashup of everything-->
<div class="top_sect wide-sect" id="real" href="http://www.youtube.com" src="one.mp3"></div>

Declare class attribute by no additional text eg: sp !red!{} to <span class="red"></span>
Declare id attribute by prefixing with # eg: sp !#red!{} to <span id="red"></span>
Declare href attribute by prefixing the link with @ eg: a !@http://google.com!{Google} to <a href="http://google.com">Google</a>
Declare src attribute by prefixing source with > eg: img !>one.png! to <img src="one.png">
You can declare as many attributes of the different types as long as they are separated by a comma

Jioning lines

Due to the fact that TTML produces indented code we also have a flag that allows you to join all the lines of your code at one go
To initialize put --j in a new line of your code this will join all lines until the next --j
If you want to join your whole code then insert this flag as the first line of your code
