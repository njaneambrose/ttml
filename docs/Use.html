This page shows you how to use the TTML gem

Lets get started

Download and install the ruby interpreter in your computer Download now!

If you already have it go to your command prompt and type: gem install ttml

Good! lets start, Welcome to the TTML world

For a simple startup and documentation go to the folder that the gem was installed with, go to the docs folder and initialize the simple server at the file serverBT.rb

Go to your browser and type the url http://localhost:1776/documentation or Click here!

The server at port 1776 offers documentation, how to use the ttml gem and a simple IDE to try out some TTML code

Go to your favourite Text editor and start enjoying TTML

Compile rule:

The variable/file where your TTML code is, where to output (if you do not output to a file write String.new), true|false for erb, true|false for shortcuts

1. The test method

The test method gives you some kind of REPL environment where you can test various code and see results
This is the perfect place to work before starting serious projects
Check the example below to see: 

 #The following is ruby code
 require 'ttml' #requires this file
 include TTML #load this module to avoid using TTML::
 a =<<_ #this is a simple here document you want to pass into the interpreter
 body{
  div class="container"{
 p{Hello i am just testing the waters}
  }
 }
 _
 tax = Route.new(a)#we call the Route class with parameter a
 puts tax.test #we call the method
 #everthing you can do using an actual file you can do it here even imports

2. The prototype method

The prototype method enables you to open a file with TTML code run it but this method does not write output to any file
Good if you are not comfortable writing here documents as for test method
Example below: 

 #The following is ruby code
 require 'ttml' #requires this file
 include TTML #load this module to avoid using TTML::
 tax = Route.new("test.ttml") #we call the Route class with parameter the file test.tax
 puts tax.prototype #we call the method
 #still there is no limitation feel free to explore

3. The pipe method

This method is used with kind of test method only that you write the output to a file supplied
Just good to show you this works at least some lasting result
Example below: 

 #The following is ruby code
 require 'ttml' #requires this file
 include TTML #load this module to avoid using TTML::
 a =<<_ #this is a simple here document you want to pass into the interpreter
  body{
  div class="container"{
 p{Hello i am just testing the waters}
  }
 }
 _
 tax = Route.new(a,"test.html") #we call the Route class with parameter a and test.html where the data is written
 tax.pipe #we call the method the file is created if not existing and overwritten if existing

4. The start method

This method is like a full pipe method where you can open a taxi file interpret it then write it to a supplied file
This is the professional way to go if you are working on projects
Example below: 

 #The following is ruby code
 require 'ttml' #requires this file
 include TTML #load this module to avoid using TTML::
 tax = Route.new("test.ttml","test.html") #we call the Route class with two parameters what to interpret and where to write
 tax.start #we call the method
 #you may supply different files as your desired output eg: xml,xhtml,html,... files

You can also use Watcher to compile in the start method,it automatically converts your file once it changes

 require 'ttml/watcher' #requires this file
  wt = Watcher.new("test.ttml","test.html",false,true)#the parameters here are the same as for calling the Route class
  wt.start #this starts the watcher default recompile time is 5 seconds but you can change e.g: wt.start(10) to make it recompile after 10 seconds
Once you go professoinal create a file with all the configurations to produce diffrent files in your project so that you only double click or run the file and all the files all compiled at one go
Hence you will focus on producing code and instantly looking at the product even if they are multiple files

5. Disabling Shortcuts

Shortcuts are useful as they give you faster ways to reach your target
But sometimes may turn against you, check this example of how to pass code for interpretation without shortcuts
To do this you will have to pass all the four variables required.

The variable/file where your TTML code is,where to output (if you do not output to a file write String.new),true or false for erb, true|false for shortcuts
 #for the test method
 tax = Route.new(a,String.new,false,false)
 tax.test
 #for prototype method
 tax = Route.new("test.ttml",String.new,false,false)
 tax.prototype
 #for pipe method
 tax = Route.new(a,"pipe.ttml",false,false)
 tax.pipe
 #for start method
 tax = Route.new("data.ttml","data.xml",false,false)
 tax.start

Simple Trick with the interpreter

This was not part of the original plan but it actually works
You can actually join different files with the import feature
See the simple example below: 

 #The following is ruby code
 require 'ttml' #requires this file
 include TTML #load this module to avoid using TTML::
 a =<<_ #this is a simple here document you want to pass into the interpreter
 % #this % prevent our imports from being executed
 &file.rb
 &file2.rb
 &file3.rb
 %
 _
 T = Route.new(a, "main.rb") #we call the Route class with parameter a and test.html where the data is written
 T.pipe #we call the method
 #this opens all the files supplied as imports and writes the to main.rb done! all files are joined
